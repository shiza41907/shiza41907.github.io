Describe the Internet Protocol Suite (TCP/IP) model of networking, in general terms, outlining how this protocol provides the data transfer mechanism required to establish client/server relationships.

When computers connect and transmit data between each other, they have to follow a set of rules to do so. These rules are made universally so that all computers can use them. These rules are called protocols. There are many different protocols made for different purposes. 'Internet Protocol Suite' is the term we use to refer to the protocols all together. The two most important protocols are Transmission Control Protocol (TCP) and the Internet Protocol (IP), which helps in making sure that data is delivered at the correct place without any errors.

How is the Hypertext Transfer Protocol (HTTP) used to facilitate client/server interaction?

The HTTP client (a browser is an example) intiates a HTTP request towards the server. Once the request is made,the client waits for the response. The server then processes the request by displaying the requested objects. This means that if the client makes a request for HTML pages towards the server, then the server will request the process by displaying the requested HTML pages.

Describe and illustrate the development of the Web in general terms using the Web 1.0, Web 2.0, and Web 3.0 generational paradigms.

Web 1.0, also referred to Syntactic web or the 'read-only web' is a type of web that existed in the era of 1990-2000. Web 1.0 only gave the ability for the user to read the information provided by the content producers. This web was designed to help people find better information. It is referred to as the 'read-only web' because it lacks the controls, visuals, and interacvtivity that we have on today's internet. This means that the user is unable to communicate the information back to those who produced the content. Examples of Web 1.0 include personal sites and static websites.

Web 2.0, also referred to as the social-web or the read-write web, is a type of web that took place in the era of 2000-2010 (still continues today). This type of web facitilitates interaction between the web users and sites, which, as a result, allows users to communicate with other users. Examples of this type of web includes applications like Facebook, Twitter, Youtube, etc.

Web 3.0, or the read-write-execute web, is a type of web of the era 2010 and later, which refers to the future of the web. Computers can interpret information like humans, in this era, by using Artificial Intelligence (AI) and Machine Learning. These sources allow the generation and distribution of useful content and information made for the specfic need of the user. This can include helping us learn what we are interested in, assist us in finding what we want, recognition of people, events, places, products, and movies, and it can understand the relationship between things. Examples of this include Apple's Siri, Alexa, Wolfram Alpha, etc.

What is the role of markup languages in the web?

HTML (hypertext markup language) is the most basic building block of the web, in that it defines the structure and meaning of the web content and information. Other markup languages besides HTML can be used to describe the appearance and how the information is presented of the website (i.e., CSS). It can also describe how complex features are implemented and how the web functions (i.e., JavaScript).

How are markup and scripting langauges similar? How are they different?

Scripting languages gives the script to perform some certain task. It is used to give guidance to another program or or even control another program. This connects one language to one other language as it doesn’t as a stand-alone. Markup languages a different from scripting languages in that they prepare a structure for the data or the appearance/design of the page, which doesn’t require the use of any logic or algorithm. The similar thing about these two languages is that both are building blocks for creating a website.

The different types of markup languages:

  • Standard Generalized Markup Language (SGML)
    • In the family of markup languages, this form can be best described as the 'parent'. SGML provides a way to define markup languages and set their standards for their form, stating and instructing what they can and cannot do, the elements that must be included (i.e. tags), and the basic structure of the language. Just like how a child posseses the genetic traits of their parents, other markup languages possess the structure and format rules from SGML.
  • Hypertext Markup Language (HTML)
    • Known as the 'child', or an application of SGML, Hypertext Markup language gives the ability to embed images, establish fonts, create page sections, etc. You can also add additional functions to a website with scripting languages like JavaScript.
  • Extensible Markup Language (XML)
    • A cousin to HTML and nephew to SGML, an XML markup language differs from HTML, in that, being a subset of SGML, it possesses some traits from SGML that a language like HTML does not have. For example, XML has the ability to make subsets (in other words, their own family) just like SGML, but HTML is limited to design and it does not have subsets or applications.
  • Extensible Hypertect Markup Language (XHTML)
    • A stricter, more XML based version of HTML, XHTML is basically HTML described as an XML application.
  • Dynamic Hypertext Markup Language (DHTML)
    • Rather than being a markup language, DHTML is term used to refer to the combination of features of various web development technologies that are used to help create dynamic and interactive web pages.
  • Markdown
    • Markdown is a lightweight markup language that you can use to add formatting elements to plaintext text documents using a text editor.
